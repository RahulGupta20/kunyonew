How It Works

The account-locked.html page is displayed when a user's account is temporarily locked due to security concerns, typically after multiple failed login attempts.

Security Trigger: Accounts are automatically locked after 5 consecutive failed login attempts to prevent brute-force attacks and unauthorized access.

When This Page Appears:

  • 5+ consecutive failed login attempts
  • Suspicious activity from unknown location
  • Multiple password reset requests in short time
  • Manual lock by admin or security system
  • Compromised account detection

URL Parameters

Parameter Description Example
unlocked Show unlocked success state true
reason Reason for account lock failed_attempts
duration Lock duration (future feature) 15

Key Features

1. Auto-Unlock Countdown Timer

  • 15-minute (900 seconds) countdown
  • Updates every second in MM:SS format
  • Auto-shows unlocked state when timer reaches 0:00
  • Animated pulsing lock icon

2. Three Unlock Options

  • Verify via Email: Links to verify-email.html
  • Verify via SMS: Links to verify-phone.html
  • Contact Support: Links to account-recovery.html

3. Lock Reason Display

Shows common reasons for account lock:

  • 5 consecutive failed login attempts
  • Suspicious activity from unknown location
  • Multiple password reset requests
  • Security measure for purchase protection

4. Info Carousel

3 auto-rotating slides about:

  • Account security and fraud prevention
  • Auto-unlock timer and instant verification
  • Prevention tips and best practices

5. Visual Design

  • Large animated lock icon with pulse effect
  • Countdown timer with clock icon
  • Hover effects on unlock option cards
  • Color-coded alerts (red for lock, green for unlock)

Production Integration

Backend - Detect Failed Login

// Node.js Example
const failedAttempts = await getFailedLoginCount(userId);

if (failedAttempts >= 5) {
  await lockAccount(userId, { duration: 15, reason: 'failed_attempts' });
  return res.redirect('/account-locked.html');
}

JavaScript - Console Logging

The page logs lock events for debugging:

console.log('Account Locked Page Loaded:', {
  lockReason: 'multiple_failed_attempts',
  lockDuration: '15 minutes',
  unlockTime: '2026-03-30T11:15:00Z',
  timestamp: '2026-03-30T11:00:00Z'
});

Unlock Verification Flow

// After successful email/SMS verification
await unlockAccount(userId);
res.redirect('/account-locked.html?unlocked=true');
Security Best Practices:
  • Track failed login attempts by IP and user ID
  • Implement exponential backoff (5 min, 15 min, 1 hour)
  • Send email notification when account is locked
  • Log all lock/unlock events for security audit
  • Rate limit unlock verification attempts
  • Use CAPTCHA after 2-3 failed attempts
