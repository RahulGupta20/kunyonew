How It Works

The session-expired.html page is shown when a user's session expires due to inactivity. Sessions typically expire after 30 minutes of no activity for security purposes.

Session Timeout: After 30 minutes of inactivity, users are automatically logged out and redirected to this page to protect their account from unauthorized access.

When This Page Appears:

  • User inactive for 30+ minutes
  • JWT/Session token expired on backend
  • User tries to access protected page without valid session
  • Session cookie expired or deleted
  • User manually logged out (optional redirect)

URL Parameters

Parameter Description Default Example
redirect Enable/disable auto-redirect to signin true false
return_url URL to redirect after viewing message signin.html index.html
duration How long session was active 45 minutes 2 hours

Key Features

1. Session Information Display

  • Session Duration: Total time user was logged in
  • Last Activity: When user last interacted (calculated as 30 minutes ago)
  • Expired Time: When session expired (shown as "Just now")

2. Auto-Redirect Countdown

5-second countdown before automatic redirect (can be disabled with ?redirect=false)

3. Quick Actions

  • Sign In Again: Primary button to re-authenticate
  • Clear Browser Data: Clears localStorage and sessionStorage before signing in

4. Security Features

  • Automatically clears auth tokens from localStorage
  • Preserves cart items and favorites
  • Explains why session expired
  • Tips for staying logged in longer

5. Info Carousel

3 auto-rotating slides explaining:

  • Auto-logout timer and security
  • Quick sign-in options
  • Data preservation (cart, favorites)

Production Integration

Backend Session Check

When detecting an expired session, redirect users with appropriate parameters:

// JavaScript - Detect expired session
if (response.status === 401 && response.error === 'session_expired') {
  const returnUrl = window.location.pathname;
  window.location.href = `session-expired.html?return_url=${returnUrl}`;
}

Axios/Fetch Interceptor Example

// Axios interceptor
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      const currentPage = encodeURIComponent(window.location.href);
      window.location.href = `session-expired.html?return_url=${currentPage}`;
    }
    return Promise.reject(error);
  }
);

Console Logging

The page logs session expiry data for debugging:

console.log('Session Expired:', {
  reason: 'inactivity_timeout',
  duration: '45 minutes',
  lastActivity: '2026-03-30T10:30:00Z',
  expiredAt: '2026-03-30T11:00:00Z',
  returnUrl: 'signin.html'
});
Security Best Practices:
  • Clear auth tokens (access_token, refresh_token, session_id)
  • Preserve user data (cart, favorites, preferences)
  • Log session expiry events for analytics
  • Implement sliding session (extend on activity)
  • Allow "Remember Me" for longer sessions

Session Management Tips

Recommended Session Durations:

Session Type Duration Use Case
Standard 30 minutes Default for most users
Remember Me 7-30 days Trusted device, less sensitive actions
High Security 5-15 minutes Payment pages, sensitive operations
API Token 1 hour Backend API access with refresh token

Preventing Session Expiry:

  • Implement activity tracking (mouse move, clicks, scrolls)
  • Send heartbeat requests every 5 minutes
  • Warn users 2 minutes before expiry
  • Offer "Stay Logged In" option
  • Use refresh tokens for extended access
