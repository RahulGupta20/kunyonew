How It Works

The auth-callback.html page handles OAuth redirects from Google and Apple. When users authenticate with social login, they are redirected to this page with specific URL parameters.

Important: You cannot access the callback page directly. It requires OAuth parameters in the URL to function properly.

URL Parameters

Parameter Description Example
code Authorization code from OAuth provider abc123xyz789
state Security token to prevent CSRF attacks random_state_token
provider OAuth provider (google or apple) google
error Error type if authentication failed access_denied

4-Step Authentication Flow

When authentication succeeds, the page shows a visual progress through 4 steps:

Step 1 Connecting to provider (1 second)
Step 2 Verifying credentials (2 seconds)
Step 3 Creating your account (3 seconds)
Step 4 Finalizing setup (4 seconds)
After completion, shows success message and auto-redirects to index.html after 2 seconds.

Production Integration

In production, configure your OAuth providers to redirect to:

https://yourdomain.com/auth-callback.html

Backend Integration Points

The page logs authentication data to console for backend integration:

console.log('OAuth Authentication:', {
  provider: 'google',
  code: 'authorization_code',
  state: 'state_token',
  timestamp: '2026-03-30T...'
});
