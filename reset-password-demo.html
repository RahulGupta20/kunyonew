How It Works

The reset-password.html page is accessed via a link sent to the user's email. The link contains a unique token that validates the password reset request.

Important: You cannot access the reset password page directly without a token. If you do, you'll see the "Invalid or Expired Link" error message.

Password Reset Flow:

  • User requests password reset on forgot-password.html
  • System sends email with reset link containing unique token
  • User clicks email link → arrives at reset-password.html with token
  • User enters new password and confirms
  • System validates token and updates password
  • User redirected to sign-in page

Key Features

1. Password Strength Indicator

Real-time password strength checker that evaluates:

  • Length (8+ characters required, 12+ for bonus points)
  • Lowercase letters (a-z)
  • Uppercase letters (A-Z)
  • Numbers (0-9)
  • Special characters (!@#$%^&*)

2. Password Match Validation

Confirms both password fields match before allowing submission

3. Show/Hide Password

Eye icon toggle buttons to view/hide password input

4. Token Security

Validates token on page load - shows error if missing or invalid

5. Info Carousel

3 auto-rotating slides about password security and token expiration

Production Integration

Email Link Format

When sending the password reset email, use this format:

https://yourdomain.com/reset-password.html?token={UNIQUE_TOKEN}

Backend Integration

The page logs password reset data to console:

console.log('Password Reset:', {
  token: 'user_token_here',
  newPassword: 'hashed_password',
  timestamp: '2026-03-30T...'
});
Security Notes:
  • Tokens should expire after 1 hour
  • Tokens should be one-time use only
  • Always hash passwords before storing
  • Validate token on backend before allowing password change
